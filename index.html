Projects

Data Catalog

Architect of a data ingestion/aggregation/enrichment pipeline. The pipeline ingested data from different companies and consolidated it into a single normalized view. This view could be accessed by consumers and it was fed to enrichment services that provided insights like:

  • Master product identification (duplicate and different representation linking)
  • Categorization
The pipeline was designed and implemented following a microservices architecture using Kotlin + Spring Boot, for most services, TypeScript, for on demand computation modules, Apache Spark, for data processing, Azure CosmosDB (MongoDB), for data storage, and GraphDB, as the source of truth for the inferencing and enrichment engines.
The architecture was designed to be resilient to service failures while being flexible to data schema and/or requirement changes.

Content Management System (CMS)

Architect in charge of designing and leading the implementation of a custom Content Management System. The system was used by operators to manage products displayed by a store. It was designed and implemented following these frameworks:

  • Domain Driven Design (DDD)
  • Event Sourcing (using Axon)
  • Command Query Responsibility Segregation (CQRS)
The system relied on a messaging bus (Apache Kafka) to communicate multiple microservices that followed an hexagonal architecture.
These microservices were written in Kotlin + Spring Boot or TypeScript + Express.js.
Events were stored in an SQL database and data projections on different MongoDB instances.
Operators interacted with the system through a dashboard developed using React, while state was managed using React Hooks.
Content ingestion and processing was orchestrated using state machines (XState) that triggered lambdas and/or jobs using a job manager.
